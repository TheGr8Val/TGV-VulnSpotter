# Challenge VS-001 — Flask Template Trap

## 🧩 Scenario / Escenario

A small internal tooling app lets support engineers generate custom status reports. The dev team built it fast over a weekend and shipped it to the internal network. "It's internal only," they said. "No need to overthink it."

[ES] Una pequeña aplicación interna permite a los ingenieros de soporte generar reportes de estado personalizados. El equipo la construyó rápido un fin de semana y la desplegó en la red interna. "Es solo interna", dijeron. "No hay que complicarse."

---

## 🔍 Find the vulnerability / Encuentra la vulnerabilidad

```python
from flask import Flask, request, render_template_string
import logging

app = Flask(__name__)
logging.basicConfig(level=logging.INFO)

REPORT_WRAPPER = """
<!DOCTYPE html>
<html>
<head><title>Status Report</title></head>
<body>
  <h2>Internal Status Report</h2>
  <div class="report-body">
    {user_content}
  </div>
  <footer>Generated by SupportDesk v1.2</footer>
</body>
</html>
"""

@app.route("/report", methods=["GET", "POST"])
def generate_report():
    if request.method == "POST":
        engineer_name = request.form.get("name", "Unknown")
        report_notes = request.form.get("notes", "")

        logging.info(f"Report generated by: {engineer_name}")

        # Build the final template with engineer's notes embedded
        template = REPORT_WRAPPER.format(user_content=report_notes)

        return render_template_string(template)

    return """
        <form method="POST">
            Name: <input name="name"><br>
            Notes: <textarea name="notes"></textarea><br>
            <input type="submit" value="Generate Report">
        </form>
    """

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
```
